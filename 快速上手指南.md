# 快速上手指南

这份说明写给想直接把桌宠跑起来的人。优先看第一种方法；这种方法不需要安装 Node.js，也不需要输入命令。

## 方法一：下载后直接运行

1. 打开这个项目的 GitHub 页面。
2. 点右侧或上方的 **Releases**。
3. 打开 `Desktop Pet latest`。
4. 在 **Assets** 里找下载文件。
5. 按你的电脑系统下载对应文件：
   - Windows：下载 `.exe`
   - macOS：下载 `.dmg` 或 `.zip`
   - Linux：下载 `.AppImage` 或 `.tar.gz`
6. 下载后先解压。如果里面还有压缩包，再解压一次。

## Windows

解压后优先找这些文件：

- `Desktop Pet.exe`
- `Desktop-Pet-...-win-x64.exe`

双击运行即可。桌宠会出现在屏幕右下角附近。

如果 Windows 提示不认识这个应用，选择“更多信息”，再点“仍要运行”。这是因为开源测试版没有购买商业代码签名证书。

## macOS

解压后优先找这些文件：

- `Desktop Pet.app`
- `Desktop-Pet-...-mac-arm64.dmg`
- `Desktop-Pet-...-mac-arm64.zip`

`mac-arm64` 是苹果芯片版，适合 M1、M2、M3、M4 这类 Mac。如果你用的是 Intel 芯片 Mac，建议先用下面的“从源码运行”方法。

如果是 `.dmg`，双击打开后把 app 拖到 Applications 里，或者直接双击 app 测试。

如果 macOS 提示“无法打开，因为无法验证开发者”，可以这样处理：

1. 右键点击 app。
2. 选择“打开”。
3. 再确认一次“打开”。

如果还是打不开，去“系统设置 > 隐私与安全性”，找到刚才被拦截的应用，点“仍要打开”。

## Linux

解压后优先找这些文件：

- `Desktop-Pet-...-linux-x86_64.AppImage`
- `Desktop-Pet-...-linux-x64.tar.gz`

如果是 AppImage，右键文件，打开属性，把“允许作为程序执行”打开，然后双击运行。

也可以在终端里运行：

```bash
chmod +x Desktop-Pet-*.AppImage
./Desktop-Pet-*.AppImage
```

## 怎么操作桌宠

- 拖动：按住桌宠移动鼠标。
- 换动作/表情：左键点一下桌宠。
- 特殊动作：双击桌宠。
- 安静互动：长按桌宠。
- 打招呼：鼠标靠近桌宠。
- 打开菜单：右键点桌宠。
- 看状态：右键菜单里查看“状态”。
- 换角色：右键菜单里选择“角色”。
- 调大小：右键菜单里选择 75% 到 200%。
- 切换模式：右键菜单里选择“自动表情”或“点击表情”。
- 显示时间：右键菜单里选择“实用 > 显示当前时间”。
- 专注提醒：右键菜单里选择“实用 > 开始 25 分钟专注”。
- 休息提醒：右键菜单里选择“实用 > 休息提醒”。
- 快速聊天：右键菜单里选择“对话 > 快速输入”。
- 完整聊天：右键菜单里选择“对话 > 完整聊天”。
- 对话设置：右键菜单里选择“对话 > 对话设置”，或在聊天窗口点“设置”。
- 退出：右键菜单里点“退出”。

## 怎么开启聊天

聊天功能需要先准备一个 OpenAI-compatible 的 LLM 接口。可以是本机模型服务，也可以是云端模型服务。

1. 右键桌宠。
2. 想像桌宠陪伴一样随手说话，就选“对话 > 快速输入”。
3. 想看聊天记录，就选“对话 > 完整聊天”。
4. 点聊天窗口右上角的“设置”，或者右键选择“对话 > 对话设置”。
5. 先在“人物设定”里填角色称呼、性格、说话方式和背景。这里会直接影响大模型的回复口吻。
6. 填入 `Base URL`，例如本机 Ollama 常见地址是 `http://localhost:11434/v1`。
7. 填入模型名。
8. 如果服务需要密钥，就填 `API key`；如果是本机模型，一般可以不填。
9. 点“测试 LLM”。如果显示连接成功，说明桌宠已经能真实调用这个模型。
10. 点“保存设置”，然后回到聊天窗口或快速输入框发送消息。

桌宠会把回复显示在气泡里，并根据对话内容切换表情。API key 会优先用系统安全存储保存在本机；如果当前系统不支持安全持久化，重启后需要重新填写。

## 怎么填写人物设定

第一次打开应用时会自动弹出“对话设置”。这里的人物设定不是装饰，它会被写进 LLM 的 system prompt。

- 角色称呼：模型会认为自己是谁，比如“露娜”“桌宠”“学习助手”。
- 性格：写角色的核心性格，比如“礼貌、聪明、稍微傲娇，但内心关心使用者”。
- 说话方式：写语气，比如“回复简短自然，偶尔吐槽，但不要刻薄”。
- 背景设定：写角色和使用者的关系、所在场景，可以留空。
- 额外规则：写你不希望角色做的事，比如“不要主动报好感数值”。

下面的“互动数值”可以设置好感度系统：

- 不想要好感度，就关闭“启用好感度系统”。
- 想改名字，可以把“好感”改成“亲密度”“信赖”等。
- 点击、聊天、双击、长按分别可以设置增加多少数值。
- 连续点击消耗活力，数值越高，频繁点她越容易进入不高兴状态。
- 开心阈值越低，角色越容易进入开心状态。

## 怎么开启语音朗读

语音朗读默认关闭。

1. 右键桌宠。
2. 选择“对话 > 对话设置”。
3. 在 TTS 区域选择语音后端：
   - 系统语音：直接用电脑自带声音。
   - GPT-SoVITS 本地推理服务：先在本机启动 GPT-SoVITS 的 API 服务，再填入服务地址，比如 `http://127.0.0.1:9880/tts`。
   - 自定义 TTS 接口：填入你自己的 TTS 接口地址，模板里 `{{text}}` 会替换成要朗读的文字。
4. 如果 TTS 服务需要密钥，就填 `TTS API key`；本地服务一般可以不填。
5. 点“测试 TTS”。如果能听到测试音频，说明桌宠已经能真实调用这个语音后端。
6. 点“保存设置”。

GPT-SoVITS 是本地训练和本地推理模型。桌宠这里填的“地址”指的是你在本机启动的推理服务 HTTP 地址，不是要求你使用云端服务。桌宠不会训练模型，只负责把要朗读的文字发给已经启动的 GPT-SoVITS 推理服务，并播放返回的音频。

如果使用官方 GPT-SoVITS 推理服务，通常还需要填写：

- 参考音频路径：GPT-SoVITS 服务能访问到的音频文件路径。
- 参考文本：参考音频对应的文字。
- 文本语言和参考文本语言：中文通常填 `zh`。
- 音频格式：建议先用 `wav` 或 `ogg`。

自定义 TTS 接口可以返回音频流，也可以返回 JSON。JSON 里可以放 `audioDataUrl`、`audio_url`、`url`、`audio` 或 `audio_base64`。

## 没看到桌宠怎么办

先看屏幕右下角附近。桌宠窗口是透明的，而且体积比较小。

还可以试试：

- 重新运行一次应用。
- 检查是不是被其他窗口挡住了。
- 在右键菜单里选“回到屏幕右下角”。
- 如果完全打不开，换用下面的“从源码运行”方法，可以看到错误提示。

## 方法二：从源码运行

这种方法适合想自己改代码、改素材，或者直接下载版本打不开的人。

1. 安装 Node.js，建议安装 LTS 版本。
2. 下载这个项目源码。
3. 打开终端，进入项目文件夹。
4. 运行：

```bash
npm install
npm start
```

如果能看到桌宠，说明运行成功。

## 自己打包

如果想在自己的电脑上生成安装包，运行：

```bash
npm run build
```

本地通常只能稳定生成当前系统对应的平台包。Windows、macOS、Linux 三端包会由 GitHub Actions 分别生成。
