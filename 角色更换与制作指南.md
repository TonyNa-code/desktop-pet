# 角色更换与制作指南

这份说明写给想换成自己角色的人。最简单的方式是先复制一个现有角色包，再替换图片和名称。

## 直接切换内置角色

运行应用后，右键点击桌宠，选择 `角色`，再选择想使用的角色即可。

如果菜单里没有新角色，通常是角色包文件夹没有放对位置，或者 `character.json` 写错了。

## 新增一个角色包

角色包都放在：

```text
assets/characters/
```

推荐做法：

1. 复制 `assets/characters/default/`。
2. 把复制出来的文件夹改名，例如 `my-character`。
3. 替换里面的 `sprite.png` 和 `preview.png`。
4. 打开 `character.json`，修改 `id`、`name`、`description`。
5. 运行 `npm run validate:assets` 检查素材。
6. 重启应用，在右键菜单 `角色` 里选择新角色。

文件结构应该像这样：

```text
assets/characters/my-character/
  character.json
  preview.png
  sprite.png
```

`id` 建议只用小写英文、数字和短横线，例如 `my-character`。不要在 `character.json` 里写本机绝对路径。

## 图片规格

`sprite.png` 是一个透明背景的动作表，默认规格：

- 每个小格：`768 x 832`
- 每行：`8` 格
- 图片宽度：`6144`
- 图片高度：`832 x 行数`
- 未使用的小格保持透明
- 角色尽量居中，脚底位置尽量一致
- 每个动作里的角色大小、发色、服装、光照要保持一致

默认 8 行动作顺序：

1. `idle`：待机
2. `runningRight`：向右移动
3. `runningLeft`：向左移动
4. `waving`：挥手
5. `jumping`：开心/跳起
6. `failed`：失败/困扰
7. `running`：通用移动
8. `review`：思考/查看

如果只想先做静态角色，也可以每行只放 1 帧，然后在 `character.json` 里把对应动作的 `frames` 改成 `1`。

## character.json 怎么改

最小示例：

```json
{
  "schemaVersion": 1,
  "id": "my-character",
  "name": "My Character",
  "description": "A custom desktop pet character.",
  "sprite": "sprite.png",
  "preview": "preview.png",
  "frame": {
    "width": 768,
    "height": 832
  },
  "columns": 8,
  "states": {
    "idle": { "row": 0, "frames": 6, "interval": 900 },
    "waving": { "row": 3, "frames": 4, "interval": 160 },
    "jumping": { "row": 4, "frames": 5, "interval": 140 },
    "failed": { "row": 5, "frames": 4, "interval": 220 },
    "review": { "row": 7, "frames": 4, "interval": 500 }
  },
  "automaticActions": ["review", "waving", "jumping"],
  "clickActions": ["waving", "review", "jumping", "failed"],
  "staticExpressions": [
    { "state": "idle", "frame": 0 },
    { "state": "waving", "frame": 1 },
    { "state": "review", "frame": 0 },
    { "state": "failed", "frame": 0 }
  ]
}
```

字段含义：

- `row`：动作在第几行，从 `0` 开始。
- `frames`：这一行动作用几帧。
- `interval`：每帧间隔，单位是毫秒，数值越小动作越快。
- `automaticActions`：自动表情模式会随机播放的动作。
- `clickActions`：点击表情模式会按顺序切换的动作。
- `staticExpressions`：点击表情用到的静态表情帧。

## AI 生图提示词模板

下面的模板适合拿去生成原创角色素材。请只生成你有权使用和发布的角色，不要直接使用没有授权的商业角色素材。

### 角色设定图

```text
Create an original chibi anime desktop pet character design.
Style: clean Japanese anime illustration, cute but polished, no pixel art.
Character: [写角色外貌，例如 silver white hair, black ribbons, red eyes, elegant school-style outfit].
Personality: [写性格，例如 proud but caring, slightly tsundere, refined and calm].
Requirements: full body, front-facing, consistent proportions, clear silhouette, simple readable details, transparent or plain white background, no text, no watermark.
Avoid: inconsistent costume, extra accessories, cropped feet, blurry image, realistic photo style, heavy shadows.
```

### 默认 8 行动作表

```text
Create a transparent-background spritesheet for an original chibi anime desktop pet.
Canvas layout: 8 columns x 8 rows. Each cell is 768 x 832. Keep every frame centered in its cell.
Character must stay consistent across all cells: same hair color, same hairstyle, same eye color, same outfit, same body proportions, same lighting, same line art style.
No text, no watermark, no background, no shadows outside the character.

Rows:
1. idle: 6 subtle breathing/blinking frames, last 2 cells transparent.
2. runningRight: 8 small walking/running frames facing right.
3. runningLeft: 8 small walking/running frames facing left.
4. waving: 4 cute waving frames, last 4 cells transparent.
5. jumping: 5 happy jumping frames, last 3 cells transparent.
6. failed: 4 embarrassed or troubled frames, last 4 cells transparent.
7. running: 6 energetic movement frames, last 2 cells transparent.
8. review: 4 thinking/checking frames, last 4 cells transparent.

Character design:
[把角色设定粘贴到这里]
```

### 更傲娇的表情动作

```text
Generate additional chibi anime desktop pet frames for the same character.
Keep exactly the same character design, outfit, colors, proportions, and line art.
Transparent background, centered full body, 768 x 832 per frame.

Expressions and actions:
- proud smile with arms crossed
- slightly annoyed pout
- embarrassed blush looking away
- happy small wave
- surprised face
- thinking pose

Avoid: changing hair color, changing outfit, changing body size, black or broken fingers, blurry edges, pixel art, inconsistent lighting.
```

### 预览图

```text
Create a clean preview image for an original chibi anime desktop pet character.
Use the same character design as the spritesheet.
Show one cute standing pose, centered, transparent or plain light background.
No text, no watermark. The character should be readable at small size.
```

## 生成素材时的小技巧

- 先固定角色设定，再生成动作，不要每次重新描述一套新衣服。
- 如果工具支持参考图，把同一张角色设定图作为每次生成动作的参考。
- 如果动作分开生成，最后要把每帧放进同样大小的小格里，角色脚底和头顶位置尽量对齐。
- 发现发色、衣服、体型变了，优先重生成那一帧，不要硬贴到原图上。
- 透明边缘如果有彩边，先用干净背景重出图，再抠图。
- 完成后运行 `npm run validate:assets`，再运行 `npm start` 看实际效果。

## 常见问题

新角色不出现在菜单里：

- 检查文件夹是不是在 `assets/characters/` 下。
- 检查 `character.json` 是不是合法 JSON。
- 检查 `id` 是否为空，`states.idle` 是否存在。

动作位置跳来跳去：

- 每帧角色没有对齐。重新整理 `sprite.png`，让角色脚底和身体中心稳定。

角色边缘发脏：

- 使用透明背景 PNG。
- 避免从复杂背景里直接抠图。
- 头发、丝带、裙摆这类细边最好在生成时就要求干净边缘。

点击表情太少：

- 在 `clickActions` 里加入更多动作名。
- 在 `staticExpressions` 里加入更多 `{ "state": "...", "frame": ... }`。
