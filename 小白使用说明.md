# 小白使用说明

这份说明写给只想把桌宠跑起来的人。不会写代码也没关系，优先看第一种方法；这种方法不需要安装 Node.js，也不需要输入命令。

## 方法一：下载后直接运行

1. 打开这个项目的 GitHub 页面。
2. 点右侧或上方的 **Releases**。
3. 打开 `Desktop Pet latest`。
4. 在 **Assets** 里找下载文件。
5. 按你的电脑系统下载对应文件：
   - Windows：下载 `.exe`
   - macOS：下载 `.dmg` 或 `.zip`
   - Linux：下载 `.AppImage` 或 `.tar.gz`
6. 下载后先解压。如果里面还有压缩包，再解压一次。

## Windows

解压后优先找这些文件：

- `Desktop Pet.exe`
- `Desktop-Pet-...-win-x64.exe`

双击运行即可。桌宠会出现在屏幕右下角附近。

如果 Windows 提示不认识这个应用，选择“更多信息”，再点“仍要运行”。这是因为开源测试版没有购买商业代码签名证书。

## macOS

解压后优先找这些文件：

- `Desktop Pet.app`
- `Desktop-Pet-...-mac-arm64.dmg`
- `Desktop-Pet-...-mac-arm64.zip`

`mac-arm64` 是苹果芯片版，适合 M1、M2、M3、M4 这类 Mac。如果你用的是 Intel 芯片 Mac，建议先用下面的“从源码运行”方法。

如果是 `.dmg`，双击打开后把 app 拖到 Applications 里，或者直接双击 app 测试。

如果 macOS 提示“无法打开，因为无法验证开发者”，可以这样处理：

1. 右键点击 app。
2. 选择“打开”。
3. 再确认一次“打开”。

如果还是打不开，去“系统设置 > 隐私与安全性”，找到刚才被拦截的应用，点“仍要打开”。

## Linux

解压后优先找这些文件：

- `Desktop-Pet-...-linux-x86_64.AppImage`
- `Desktop-Pet-...-linux-x64.tar.gz`

如果是 AppImage，右键文件，打开属性，把“允许作为程序执行”打开，然后双击运行。

也可以在终端里运行：

```bash
chmod +x Desktop-Pet-*.AppImage
./Desktop-Pet-*.AppImage
```

## 怎么操作桌宠

- 拖动：按住桌宠移动鼠标。
- 换动作/表情：左键点一下桌宠。
- 特殊动作：双击桌宠。
- 安静互动：长按桌宠。
- 打招呼：鼠标靠近桌宠。
- 打开菜单：右键点桌宠。
- 看状态：右键菜单里查看“状态”。
- 换角色：右键菜单里选择“角色”。
- 调大小：右键菜单里选择 75% 到 200%。
- 切换模式：右键菜单里选择“自动表情”或“点击表情”。
- 显示时间：右键菜单里选择“实用 > 显示当前时间”。
- 专注提醒：右键菜单里选择“实用 > 开始 25 分钟专注”。
- 休息提醒：右键菜单里选择“实用 > 休息提醒”。
- 快速聊天：右键菜单里选择“对话 > 快速输入”。
- 完整聊天：右键菜单里选择“对话 > 完整聊天”。
- 对话设置：右键菜单里选择“对话 > 对话设置”，或在聊天窗口点“设置”。
- 退出：右键菜单里点“退出”。

## 怎么开启聊天

聊天功能需要先准备一个 OpenAI-compatible 的 LLM 接口。可以是本机模型服务，也可以是云端模型服务。

1. 右键桌宠。
2. 想像桌宠陪伴一样随手说话，就选“对话 > 快速输入”。
3. 想看聊天记录，就选“对话 > 完整聊天”。
4. 点聊天窗口右上角的“设置”，或者右键选择“对话 > 对话设置”。
5. 填入 `Base URL`，例如本机 Ollama 常见地址是 `http://localhost:11434/v1`。
6. 填入模型名。
7. 如果服务需要密钥，就填 `API key`；如果是本机模型，一般可以不填。
8. 点“保存设置”，然后回到聊天窗口或快速输入框发送消息。

桌宠会把回复显示在气泡里，并根据对话内容切换表情。API key 会优先用系统安全存储保存在本机；如果当前系统不支持安全持久化，重启后需要重新填写。

## 怎么开启语音朗读

语音朗读默认关闭。

1. 右键桌宠。
2. 选择“对话 > 对话设置”。
3. 在 TTS 区域选择语音后端：
   - 系统语音：直接用电脑自带声音。
   - GPT-SoVITS API：填入本地 GPT-SoVITS 服务地址，比如 `http://127.0.0.1:9880/tts`。
   - 自定义 TTS API：填入你自己的 TTS 接口地址，模板里 `{{text}}` 会替换成要朗读的文字。
4. 如果 TTS 服务需要密钥，就填 `TTS API key`；本地服务一般可以不填。
5. 点“保存设置”。

## 没看到桌宠怎么办

先看屏幕右下角附近。桌宠窗口是透明的，而且体积比较小。

还可以试试：

- 重新运行一次应用。
- 检查是不是被其他窗口挡住了。
- 在右键菜单里选“回到屏幕右下角”。
- 如果完全打不开，换用下面的“从源码运行”方法，可以看到错误提示。

## 方法二：从源码运行

这种方法适合想自己改代码、改素材，或者直接下载版本打不开的人。

1. 安装 Node.js，建议安装 LTS 版本。
2. 下载这个项目源码。
3. 打开终端，进入项目文件夹。
4. 运行：

```bash
npm install
npm start
```

如果能看到桌宠，说明运行成功。

## 自己打包

如果想在自己的电脑上生成安装包，运行：

```bash
npm run build
```

本地通常只能稳定生成当前系统对应的平台包。Windows、macOS、Linux 三端包会由 GitHub Actions 分别生成。
